What is Git?

Git is a version control system; you can think of it as a series of snapshots (commits) of your code base. With the creation of each snapshot you build out a whole application, seeing the growth of a full app. During this time you can make branches to experiment and come back to a previous snapshots you took or merge those experiments into your app.

What is GitHub?

GitHub is a code hosting platform for version control and collaboration.It lets you and others work together on projects from anywhere using those code snapshots that you took during development.

  • Git init

  • Git init is the act of registering this current code base.

  • Git add

  • Git add is how we stage the integrationof our current code base into our current list of snapshots.

  • Git status

  • Git status allows us to see the changes we are going to make.

  • Git commit

  • Git commit; finally we can commit those changes we added to our last snapshot of our code with whatever new changes were made

  • Git remote add

  • Git remote add is used to set up an online repository where we can place all of our current code. This is where he outside world can see our code base.

  • Git push

  • Git push is how we get our code to Github, an excellent place to store all of our repositories. Other developers might look at out existing projects to see how we approached problems, pull down our code and add new features or even update our code base with features they wanted to add.
